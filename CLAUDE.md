# CLAUDE.md — Лендинг консалтинговой компании

## Задача
Создай production-ready одностраничный лендинг для консалтинговой компании.
Результат — один файл `index.html` (HTML + CSS + JS внутри).
Никаких фреймворков, никаких внешних зависимостей кроме Google Fonts.
Файл должен открываться в браузере без сервера и быть готов к деплою на любой статический хостинг.

---

## Компания
- **Название:** Quant Limited
- **Специализация:** FinTech
- **Позиционирование:** Создаем ценность для общества через содействие МСБ 
- **Целевая аудитория:** Профессионалы своего дела
- **Тон:** профессиональный, но без канцелярщины — говорим на языке клиента

## Структура страницы (секции по порядку)

### 1. Header / Nav
- Логотип Название компании
- Навигация: Услуги, О нас, Кейсы, Контакты
- Кнопка CTA: "Получить консультацию"
- На мобиле — бургер-меню с плавным открытием

### 2. Hero
- Главный заголовок H1 — чёткий оффер, не абстрактный (пример: "Увеличиваем прибыль бизнеса через операционную эффективность")
- Подзаголовок — 1–2 предложения, конкретная польза
- Два CTA: основной ("Обсудить проект") и вторичный ("Смотреть кейсы")
- Визуальный акцент: геометрическая фигура, градиент или абстрактный паттерн — без стоковых фото

### 3. Цифры / Социальное доказательство
- 3–4 ключевых показателя: лет опыта, клиентов, проектов, среднее ROI
- Крупная типографика, лаконично

### 4. Услуги
- 3–6 карточек услуг
- Каждая: иконка (SVG или Unicode), название, 2–3 строки описания
- Сетка: 3 колонки на десктопе, 1 на мобиле

### 5. О компании
- Короткий текст: философия, подход, почему мы
- Без воды — конкретные принципы работы (3–4 пункта)

### 6. Кейсы / Результаты
- 2–3 карточки: клиент (анонимно или с именем), задача, результат
- Акцент на измеримый результат: "+40% выручки", "−30% операционных затрат"

### 7. Отзывы
- 2–3 цитаты клиентов: имя, должность, компания
- Простой слайдер или статичная сетка

### 8. Форма контакта
- Поля: Имя, Email, Телефон (опционально), Сообщение
- Кнопка: "Отправить заявку"
- После отправки — показать thank-you сообщение (без перезагрузки страницы)
- Валидация полей на JS

### 9. Footer
- Логотип + короткий слоган
- Навигация
- Контакты: email, телефон, город
- Копирайт
- Ссылки на соцсети (иконки SVG)

## Технические требования

### Responsive Design
- Mobile-first подход
- Breakpoints: 480px, 768px, 1024px, 1280px
- Плавные переходы между брейкпоинтами (не резкие скачки)
- Touch-friendly: минимальный размер кнопки 44px
- Навигация адаптируется в бургер-меню на мобиле

### Производительность
- Только один HTML-файл, готовый к деплою
- Шрифты: подключить через Google Fonts (не более 2 семейств)
- Все SVG-иконки — инлайн внутри HTML
- Изображения не использовать — только CSS-паттерны и градиенты
- Минимальное время загрузки

### Анимации
- Плавное появление секций при скролле (Intersection Observer)
- Hover-эффекты на кнопках и карточках
- Плавный скролл к якорям
- Анимации — только CSS-transitions, без тяжёлых библиотек

### Доступность
- Семантический HTML: header, nav, main, section, footer, h1–h3
- Alt-тексты для всех изображений
- Контраст текста соответствует WCAG AA
- Фокус-стили для клавиатурной навигации

### SEO
- Title и meta description заполнены
- Open Graph теги (og:title, og:description)
- Структурированные заголовки h1 → h2 → h3

## Дизайн

### Палитра
- **Нейтральная / деловая:** светло-серый `#F7F8FA`, угольный `#2D2D2D`, акцент — индиго `#4F46E5`

Все цвета — через CSS custom properties:
```css
:root {
  --color-bg: ...;
  --color-text: ...;
  --color-accent: ...;
  --color-accent-hover: ...;
  --color-surface: ...;
  --color-border: ...;
}
```

### Типографика
- Display / заголовки: характерный шрифт с индивидуальностью (не Inter, не Roboto)
  - Варианты: Playfair Display, Fraunces, Syne, Cabinet Grotesk, DM Serif Display
- Body: читаемый, нейтральный
  - Варианты: DM Sans, Plus Jakarta Sans, Outfit, Nunito

### Кнопки
```css
/* Primary */
background: var(--color-accent);
color: white;
padding: 14px 32px;
border-radius: 6px;
font-weight: 600;
transition: all 0.2s ease;

/* Secondary */
background: transparent;
border: 2px solid var(--color-accent);
color: var(--color-accent);
```

## Что НЕ делать
- Не использовать стоковые изображения и placeholder-картинки
- Не злоупотреблять тенями — максимум 1–2 уровня
- Не ставить больше 3 шрифтов
- Не делать анимации дольше 400ms (раздражает)
- Не забыть про состояния: hover, focus, active, disabled
- Не использовать `!important` без крайней необходимости
- Не оставлять lorem ipsum в финальном файле

## Деплой

Готовый файл `index.html` можно разместить на:
- **Netlify** — перетащить папку с файлом на netlify.com/drop
- **Vercel** — `vercel --prod` или через GitHub
- **GitHub Pages** — залить в репозиторий, включить Pages
- **Любой VPS** — положить в `/var/www/html/`, настроить nginx

Никаких серверов, баз данных и сборщиков не требуется — это статика.
